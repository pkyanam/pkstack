# templates/web ‚Äî AGENTS.md

## Purpose

Source-of-truth Next.js 15 template for `npm create pkstack`. This is exactly what a scaffolded project looks like. Every change here must pass `tsc --noEmit` and `next build`.

## Public API

This template is not a published package ‚Äî it is the scaffold output. The "API" is the file structure and conventions that the CLI copies:

- **`src/db/schema.ts`** ‚Äî app Drizzle tables (edit to add your own tables)
- **`src/db/auth-schema.ts`** ‚Äî Better Auth tables (DO NOT EDIT ‚Äî regenerate from CLI)
- **`src/db/index.ts`** ‚Äî exports `db` (Drizzle instance)
- **`src/lib/auth.ts`** ‚Äî exports `auth` (Better Auth instance)
- **`src/server/api/trpc.ts`** ‚Äî exports `createTRPCRouter`, `publicProcedure`, `protectedProcedure`
- **`src/server/api/root.ts`** ‚Äî exports `appRouter` and `AppRouter` type
- **`src/server/api/server.ts`** ‚Äî exports `api` for RSC server-side tRPC calls
- **`src/server/api/provider.tsx`** ‚Äî exports `TRPCProvider` and `trpc` client
- **`scripts/check-env.ts`** ‚Äî validates required env vars before dev/build

## Do Not Modify

- **`src/db/auth-schema.ts`** ‚Äî generated by `npx auth@latest generate`. Regenerate, don't hand-edit.
- The tRPC/React Query setup pattern in `query-client.ts` and `provider.tsx` ‚Äî it follows the official tRPC App Router setup guide exactly. Deviating causes hydration mismatches.
- The `scripts/check-env.ts` exit codes ‚Äî exit 0 = ok, exit 1 = missing vars. The npm `dev`/`build` scripts depend on this behavior.

## Common Agent Mistakes

1. **Adding business logic to `auth.ts`** ‚Äî `src/lib/auth.ts` is only the Better Auth instance config. Business logic (user profile, roles, etc.) goes in tRPC procedures, not here.
2. **Importing from `src/server/api/server.ts` in Client Components** ‚Äî `server.ts` is RSC-only (it imports `server-only`). Use the `trpc` client from `provider.tsx` in Client Components instead.
3. **Skipping the migration step after schema changes** ‚Äî after editing `schema.ts`, you must run `npm run db:generate` then `npm run db:migrate`. Skipping generates TypeScript errors that look like schema mismatches.
