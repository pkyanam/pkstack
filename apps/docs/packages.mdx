# Packages

pkstack is built around explicit package ownership.

## `@pkstack/ui`

Owns shared React UI primitives.

Examples:

- `Button`
- `buttonVariants`
- `cn`

This package owns reusable presentation primitives, not app state or business logic.

## `@pkstack/db`

Owns shared database helpers.

Examples:

- `createDb`
- `createPostgresClient`
- `mergeSchemas`
- Drizzle config helpers

This package owns database wiring, not the auth schema definition.

## `@pkstack/auth`

Owns Better Auth schema and helper wiring.

Examples:

- `authSchema`
- `createAuth`
- `getSession`
- route-handler helpers

This package owns the auth contract, but apps still pass in their db instance.

## `@pkstack/ai`

Owns thin AI SDK helpers and reusable utilities.

Examples:

- `streamText`
- `useChat`
- `createAI`
- message trimming helpers

This package does not own persistence or provider-specific app logic.

## `@pkstack/api`

Owns plain shared contracts.

Examples:

- zod schemas
- input/output types
- router contract types

This package must stay framework-agnostic.

## `@pkstack/config`

Owns shared repo and scaffold tooling defaults.

Examples:

- strict tsconfig presets
- shared ESLint config
- Tailwind preset
- `require-variants` lint rule

## Ownership rule

If a concern is reused across scaffolded apps, it probably belongs in a package.

If a concern is specific to one scaffolded app’s feature logic, it belongs in the template or generated app instead.
