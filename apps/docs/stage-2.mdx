# Release State

`v0.2.0` is the package extraction release.

## What shipped

- published `create-pkstack`
- published `@pkstack/config`
- published `@pkstack/ui`
- published `@pkstack/db`
- published `@pkstack/auth`
- published `@pkstack/ai`
- published `@pkstack/api`
- web and mobile source-of-truth templates
- Expo reference app
- Mintlify docs content
- Playwright scaffold E2E coverage

## What the release changed

Before `v0.2.0`, core runtime pieces lived mostly inside the web template.

After `v0.2.0`:

- shared runtime code lives in packages
- templates stay thinner and more app-owned
- the CLI scaffolds apps that depend on published pkstack packages

## What is next

The next release step is not another architecture change.

It is release validation:

- ship the web scaffold type fix in `v0.2.2`
- re-run the published web smoke test after that release is live
